# 產品需求文件 (PRD) - 個人記帳簿系統

## 1. 專案概述

### 背景與動機
現代人生活忙碌，常常不清楚自己的金錢流向，導致月底容易透支或無法達成儲蓄目標。因此，我們需要一個簡單易用的個人記帳系統，幫助使用者快速記錄與分析每日收支，進而建立良好的財務習慣。

### 目標用戶
一般大眾，任何想要追蹤個人財務狀況、控制預算並養成記帳習慣的人。

### 核心價值主張
一個幫助使用者快速記錄與分析收支的個人記帳系統，提供直覺的介面與清晰的統計報表，讓財務管理變得輕鬆無負擔。

---

## 2. 功能需求

### 2.1 收支快速記錄
- **使用者故事**：作為一般使用者，我希望能夠快速輸入收支的金額、日期、類別與備註，以便在花錢後能立即記錄，避免遺忘。

### 2.2 類別分類統計
- **使用者故事**：作為一般使用者，我希望系統能將我的收支按類別（如餐飲、交通、娛樂等）進行分類統計，以便我了解資金的主要流向。

### 2.3 月度收支總覽
- **使用者故事**：作為一般使用者，我希望能在一個總覽頁面看到每個月的總收入、總支出與結餘，以便掌握整體的財務健康狀況。

### 2.4 搜尋與篩選功能
- **使用者故事**：作為一般使用者，我希望能夠透過日期範圍、特定類別或關鍵字來搜尋與篩選過去的記帳紀錄，以便快速尋找特定的歷史花費。

### 2.5 預算提醒
- **使用者故事**：作為一般使用者，我希望可以設定每月的預算上限，並在支出接近或超過預算時收到視覺上的提醒，以便我控制開銷。

---

## 3. 非功能需求

### 3.1 技術限制
- 後端框架：Flask
- 模板引擎：Jinja2
- 資料庫：SQLite
- 網頁前端：HTML, CSS (Vanilla CSS), JavaScript

### 3.2 效能與安全考量
- **效能**：系統操作應保持快速響應，頁面載入時間與資料新增/查詢操作不應有明顯延遲。
- **安全**：雖然是個人記帳系統，但仍需防止基本的網頁攻擊（如 SQL Injection、XSS），確保使用者資料的完整性。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備)
- 收支快速記錄（新增、編輯、刪除紀錄）
- 月度收支總覽（首頁顯示當月總收支與結餘）
- 基礎的類別設定（內建常見分類供選擇）

### Should Have (應該具備)
- 類別分類統計（圖表顯示各類別花費比例）
- 搜尋與篩選功能（依照日期或類別過濾明細）

### Nice to Have (有了更好)
- 預算提醒（設定預算並顯示達標進度條）
- 自訂收支類別（允許使用者新增或修改類別）
